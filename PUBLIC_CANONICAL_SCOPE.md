# PUBLIC_CANONICAL_SCOPE

## Purpose
Define what qualifies as public canonical versus restricted canonical doctrine.

## Public Canonical
Public canonical doctrine is:
- field-defining
- conceptual
- implementation-agnostic
- stable under ontology governance

Examples:
- runtime legitimacy
- authority continuity
- mutation legitimacy
- bounded execution
- temporal legitimacy
- runtime divergence
- legitimacy revalidation
- authority externalization

## Restricted Canonical
Restricted canonical doctrine is implementation-adjacent and substrate-revealing.

Examples:
- replay implementation details
- verification mechanics
- execution semantics
- orchestration logic
- operational constraints
- deployment assumptions

## Decision Rule
If content helps explain field ontology only, classify as public canonical.
If content helps reconstruct implementation behavior, classify as restricted canonical.
