# Public Surface Definition

## Canonical Public Scope
The public repository may contain only:
- `docs/notebooklm_public/`
- conceptual doctrine and legitimacy field theory
- category framing and conceptual comparisons
- public-safe demo experience (`demo/`)
- public assets/examples that do not expose implementation mechanics

## Allowed Public Layers
- foundational conceptual doctrine
- canonical conceptual invariants
- formal vocabulary and conceptual state models
- conceptual topology, emergence, and comparison models
- philosophical and cross-domain conceptual framing
- public-safe contradiction demo and evidence interpretation surface

## Forbidden Public Layers
- runtime substrate
- orchestration logic
- execution topology internals
- replay implementation details
- enforcement semantics and decision internals
- deployment/runtime private architecture
- operational sequencing and factory workflow
- implementation-sensitive code paths

## Public Boundary Test
Question:
Does this artifact help explain legitimacy theory, or does it reveal operational implementation?

Decision:
- conceptual explanation only: allowed
- implementation reconstruction possible: forbidden

## Surface Ownership
- Public repository owners enforce conceptual boundary.
- Private repository owners maintain operational implementation.
- Cross-boundary materials require disclosure review before publication.
