Understanding this demo

          
This demo is intentionally minimal.

          
It is not showcasing a smart AI agent.
It is not a cybersecurity dashboard.
It is not an orchestration platform.

          
Its purpose is to expose a hidden assumption most autonomous systems still rely on:

          
If a task was approved once, can it still execute safely after the system itself has changed?

          
The runtime snapshots represent the system state at two different moments:

          
    
            
  • when approval was granted
    • 
            
  • when execution resumed
    • 
          

          
You do not need to understand every technical field.

          
What matters is this:

          
The system reality is no longer identical.

          
The demo intentionally shows execution continuing under changed runtime conditions.

          
This is the contradiction DETERMA focuses on.

          
Traditional flow:
Approval → Execution

          
DETERMA:
Approval → Runtime Revalidation → Execution Decision

          
At the end of the flow you can open the execution evidence trail.

          
The evidence can be copied into ChatGPT, Claude or Gemini for independent reconstruction.
