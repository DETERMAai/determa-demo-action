# DETERMA Investor Onboarding Flow

## Purpose

This document defines the recommended onboarding sequence for investors reviewing DETERMA.

The objective is to:

- reduce cognitive overload
- preserve architectural clarity
- prevent accidental overexposure
- guide understanding progressively

---

# Recommended Review Sequence

## Step 1 — Public Showcase

Objective:

Understand the category and problem.

Suggested materials:

- public repository
- public README
- split demonstration
- public architecture overview
- presentation material

---

## Step 2 — Curated Diligence

Objective:

Understand the runtime philosophy and governance model.

Suggested materials:

- sanitized runtime success criteria
- sanitized threat scenarios
- sanitized MVP architecture
- design partner overview
- selected governance philosophy documents

---

## Step 3 — Guided Technical Review

Objective:

Validate technical seriousness.

Possible topics:

- execution legitimacy
- replay-aware governance
- runtime lineage
- constrained mutation
- fail-closed runtime behavior

Review should remain guided.

---

## Step 4 — Strategic Technical Diligence

Objective:

Support deeper strategic conversations where appropriate.

This stage may include:

- selected internal walkthroughs
- deeper runtime discussions
- controlled architecture review

Only after explicit approval.

---

# Important Principle

The onboarding sequence should move:

```text
from understanding the problem
→ to understanding the architecture direction
→ to understanding the runtime philosophy
→ to controlled technical depth
```

not directly into implementation internals.

---

# Canonical Principle

```text
Progressive understanding is safer and more effective than unrestricted exposure.
```
