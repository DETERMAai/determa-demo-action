# Public and Diligence NotebookLM Corpus Guide

## Purpose

This document defines how NotebookLM corpora should be constructed for DETERMA.

The objective is to:

- maximize understanding
- minimize implementation exposure
- preserve strategic moat boundaries
- support guided technical diligence

---

# Public NotebookLM Corpus

## Allowed Sources

The public NotebookLM corpus may contain:

- public README material
- public architecture overviews
- demo walkthroughs
- FAQ material
- public-safe onboarding
- sanitized presentations
- split demonstration framing
- public-safe threat framing

---

## Forbidden Sources

The public NotebookLM corpus must not contain:

- patent drafts
- cryptographic specifications
- replay invalidation semantics
- authority issuance mechanics
- deep governance internals
- orchestration internals
- implementation invariants
- internal sequencing strategy
- unrestricted architecture archives

---

# Curated Diligence NotebookLM Corpus

## Intended Audience

- serious investors
- enterprise evaluators
- design partners
- strategic technical reviewers

---

## Allowed Sources

The curated diligence corpus may additionally include:

- selected runtime success criteria
- selected threat scenarios
- sanitized MVP architecture
- selected governance philosophy docs
- selected runtime diagrams
- selected runtime proof artifacts

---

## Still Restricted

Even during diligence, do not include:

- invention-sensitive material
- patent drafts
- cryptographic internals
- deep replay semantics
- capability lifecycle internals
- governance mutation mechanics
- moat-critical sequencing

---

# Canonical Principle

```text
NotebookLM should explain the system.
It should not expose the moat.
```
