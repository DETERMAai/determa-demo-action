# DETERMA Public Positioning Canon

## Purpose

This document defines the canonical public framing for DETERMA.

The objective is to:

- preserve messaging consistency
- establish category ownership
- prevent overexposure
- align public communication with the disclosure doctrine

---

# Canonical Public Definition

```text
DETERMA is a governed execution boundary for machine-initiated change.
```

---

# Core Public Problem

As AI systems become capable of:

- modifying code
- changing infrastructure
- triggering workflows
- operating production systems

organizations require mechanisms that govern whether execution should occur.

---

# Public Framing Principles

## Emphasize

- execution legitimacy
- fail-open vs fail-closed execution
- governed execution
- replay-aware runtime philosophy
- constrained mutation
- runtime trust boundaries
- operational accountability

---

## Avoid Overexposure

Do not publicly explain:

- exact replay invalidation mechanics
- authority issuance semantics
- capability lifecycle internals
- governance mutation mechanics
- cryptographic enforcement details
- moat-critical sequencing

---

# Public Narrative Direction

Public communication should explain:

```text
why execution governance matters
```

not:

```text
exactly how the moat works internally
```

---

# Canonical Messaging Themes

## Theme 1 — Execution Governance

AI-generated intent should not automatically receive mutation authority.

---

## Theme 2 — Runtime Legitimacy

Execution legitimacy should be evaluated at execution time.

---

## Theme 3 — Fail-Closed Runtime Philosophy

Under uncertainty, execution should halt rather than proceed implicitly.

---

## Theme 4 — Replay Awareness

Historical authority should not automatically remain valid in changed runtime conditions.

---

# Canonical Principle

```text
The public layer should make the problem obvious.
The private layer should preserve the mechanism.
```
