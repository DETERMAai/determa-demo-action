# DETERMA NotebookLM Disclosure Policy

## Purpose

This document defines the canonical policy for constructing NotebookLM and AI-assisted diligence corpora.

NotebookLM systems should be treated as disclosure surfaces.

The objective is to:

- support guided understanding
- reduce cognitive overload
- preserve moat boundaries
- prevent accidental implementation leakage

---

# Canonical Principle

```text
NotebookLM should explain the system.
It should not expose the moat.
```

---

# Allowed NotebookLM Usage

NotebookLM may be used for:

- investor onboarding
- design partner onboarding
- technical orientation
- runtime philosophy explanation
- governance education
- public-safe architecture walkthroughs

---

# Public NotebookLM Corpus

## Allowed Sources

- public README material
- public architecture overviews
- public-safe onboarding
- sanitized runtime walkthroughs
- sanitized threat framing
- public FAQ
- design partner overview

---

## Forbidden Sources

- patent drafts
- cryptographic internals
- replay invalidation mechanics
- authority issuance semantics
- deep governance internals
- orchestration internals
- moat-critical sequencing
- unrestricted architecture archives

---

# Curated Diligence NotebookLM Corpus

## Allowed Additional Sources

- sanitized runtime success criteria
- sanitized MVP architecture
- selected threat scenarios
- selected runtime proof artifacts
- selected governance philosophy docs

---

## Still Restricted

Even in diligence corpora, do not expose:

- invention-sensitive internals
- governance mutation semantics
- capability lifecycle mechanics
- deep replay semantics
- restricted runtime invariants
- sensitive operational sequencing

---

# Operational Rules

## Rule 1

Do not ingest unrestricted internal repositories into public-facing NotebookLM systems.

---

## Rule 2

Always use curated corpora.

---

## Rule 3

Treat retrieval systems as potential architectural exposure surfaces.

---

# Strategic Principle

```text
Guided understanding creates trust.
Unrestricted exposure destroys boundaries.
```
