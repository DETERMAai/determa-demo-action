# DETERMA Public vs Private Disclosure Doctrine

## Purpose

This document defines the canonical disclosure boundaries for DETERMA.

Its purpose is to:

- preserve strategic clarity
- protect implementation depth
- prevent accidental IP exposure
- maintain category leadership
- separate proof from mechanism
- standardize external disclosure decisions

This doctrine applies to:

- repositories
- demos
- presentations
- NotebookLM corpora
- investor materials
- partner onboarding
- public documentation
- conference talks
- technical walkthroughs
- AI-assisted diligence systems

---

# Core Strategic Principle

```text
Public = proof.
Private = mechanism.
```

Public systems should demonstrate:

- the problem
- the category
- the architectural direction
- the runtime philosophy
- the governance model at a high level
- the operational outcome

Private systems should contain:

- implementation semantics
- protocol mechanics
- runtime invariants
- capability lifecycle details
- replay invalidation internals
- governance mutation semantics
- cryptographic architecture
- sequencing strategy
- patent-sensitive material

# Canonical Disclosure Test

Before exposing any material externally, ask:

## Question 1

Does this explain:

```text
what the system achieves?
```

or:

```text
how the moat works internally?
```

If it exposes moat mechanics, it should remain private.

---

## Question 2

Could a sophisticated competitor use this material to:

- shortcut architectural learning
- replicate sequencing
- infer hidden invariants
- reproduce protocol primitives
- compress implementation time

If yes, restrict disclosure.
