# DETERMA Source of Truth Policy

## Purpose

This document defines how canonical truth is maintained across DETERMA repositories, archives, diligence layers, and NotebookLM corpora.

The objective is to:

- prevent architectural drift
- reduce duplicate truths
- preserve continuity
- establish authoritative ownership boundaries
- support long-term governance integrity

---

# Canonical Principle

```text
Every important concept should have one canonical source of truth.
```

Derivative materials may exist.

Canonical ownership should remain singular.

---

# Canonical Truth Layers

| Layer | Role |
|---|---|
| Internal Core | Canonical authority source |
| Internal Archive | Canonical restricted preservation |
| Curated Diligence | Sanitized derivative layer |
| Public Showcase | Simplified external proof layer |

---

# Derivative Material Rules

Sanitized or public-safe documents:

- are derivative artifacts
- are not canonical implementation sources
- should reference canonical origins where appropriate

# NotebookLM Rule

NotebookLM corpora are derivative understanding systems.

They are not canonical memory systems.

---

# Migration Rule

When restructuring:

- move originals carefully
- preserve references
- preserve lineage
- avoid duplicate canonical ownership

---

# Anti-Drift Rule

When two documents conflict:

- canonical source overrides derivative source
- derivative material must be updated
