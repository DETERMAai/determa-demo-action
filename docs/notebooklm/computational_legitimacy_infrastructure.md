# Computational Legitimacy Infrastructure

AI systems require legitimacy infrastructure, not only intelligence infrastructure.

As model capability rises, mutation surfaces expand. The limiting factor becomes not idea generation, but whether execution remains legitimate under current runtime conditions.

## Core Components (Conceptual)

- execution legitimacy: runtime-time validity of a mutation
- runtime trust continuity: persistence of decision relevance across time
- mutation containment: bounded state-change authority
- authority continuity: stable control boundaries from proposal to execution
- replay-aware governance: treatment of repeated execution under changed context
- legitimacy checkpoints: explicit decision gates at mutation time

## Why Intelligence Infrastructure Alone Is Insufficient

Intelligence infrastructure improves proposal quality.
Legitimacy infrastructure constrains mutation consequences.

Both are required for reliable autonomous mutation environments.

## Category Positioning

This is not security theater and not generic workflow tooling.

DETERMA is positioned as computational legitimacy infrastructure:
- focused on whether execution still belongs to current runtime reality
- focused on continuity between authorization and mutation
- focused on preventing stale-legitimacy execution

## Practical Realization

When systems evolve continuously, legitimacy must be treated as a first-class runtime primitive.
