# DETERMA NotebookLM FAQ

## Why does runtime drift matter?
Runtime systems evolve continuously, so authorization assumptions can become outdated.

## Why can approval become stale?
Approval is tied to a historical context; execution occurs in present context.

## Why must legitimacy be re-evaluated?
Because execution authority must match current runtime reality.

## What changes between approval and execution?
Repository state, dependencies, and runtime environment can change over time.

## Why govern execution instead of prompts?
Prompts govern intent quality; governed execution governs mutation authority at runtime.
