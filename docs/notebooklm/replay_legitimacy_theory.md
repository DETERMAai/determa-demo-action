# Replay Legitimacy Theory

Replayability alone is insufficient without legitimacy continuity.

A replay can be deterministic and still illegitimate if runtime conditions have diverged from the original authorization context.

## Replay Versus Legitimacy

Replay answers: "Can this action be repeated?"
Legitimacy answers: "Should this action execute now?"

These are distinct dimensions.

## Deterministic Replay Boundaries

Determinism can preserve reproducibility of action structure. It does not guarantee equivalence of execution context.

## Runtime State Dependence

Mutation outcomes and risk profiles depend on current runtime state. If state continuity breaks, historical authorization must not be interpreted as present legitimacy.

## Replay-Aware Authorization

Replay-aware authorization treats repeated execution as a new legitimacy question under current runtime conditions.

## Historical Replay Invalidation

Historical authorization may remain valid as a record while becoming invalid as execution authority. That distinction is foundational to runtime legitimacy reasoning.
