# Excluded From Ingestion

The following categories are intentionally excluded to maximize reasoning quality and reduce governance noise:

- meta-governance layers
- cognition-governance maintenance layers
- disclosure governance and sterilization policy docs
- repository hygiene and release governance docs
- topology-governance administration docs
- ontology-maintenance-only controls
- operational governance artifacts
- implementation-adjacent mechanics
- runtime substrate references
- orchestration references
- replay implementation semantics

## Why Exclusion Improves Quality
- reduces ontology clutter
- avoids policy-over-theory weighting
- keeps NotebookLM focused on conceptual field reasoning
- prevents implementation gravity from distorting public doctrine synthesis
