# Canonical Concept Hierarchy

```text
LEGITIMACY
├─ Runtime Legitimacy
│  ├─ Authority Continuity
│  ├─ State Continuity
│  ├─ Temporal Validity
│  ├─ Replay Legitimacy
│  └─ Legitimacy Revalidation
├─ Mutation Legitimacy
├─ Execution Authority
│  └─ Bounded Execution
├─ Runtime Divergence
│  ├─ Drift Accumulation
│  └─ Mutation Reinterpretation
└─ Computational Invalidation
```

Parent-child interpretation:
- Parent concepts define admissibility classes.
- Child concepts define mechanisms of preservation or breakdown.

Boundary rule:
If a concept does not affect execution admissibility under current state, it is adjacent context, not core hierarchy.
