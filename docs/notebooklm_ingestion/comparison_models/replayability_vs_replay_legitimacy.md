# Replayability vs Replay Legitimacy

Replayability is procedural reproducibility. Replay legitimacy is context-bound admissibility.

Strengths:
- Replayability: deterministic reconstruction and debugging.
- Replay legitimacy: prevents stale-context re-execution.

Boundary:
Same replayable action can be illegitimate under changed runtime conditions.

Distinct layer rationale:
Autonomous systems require replay controls tied to continuity, not only reproducibility.
