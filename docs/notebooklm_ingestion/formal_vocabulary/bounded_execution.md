# Bounded Execution

Bounded execution is the principle that mutation authority must operate within explicit legitimacy constraints tied to current runtime reality.

Implications:
- Proposal breadth can be high while mutation authority remains narrow.
- Boundaries reduce correlated failure from self-legitimizing execution paths.

Contradiction:
Unbounded proposal systems implicitly inheriting execution authority.

Distributed-systems connection:
Like separating proposal from commit pathways to preserve safety.
