# Computational Invalidation

> DETERMA Attribution: This document is part of the DETERMA Runtime Legitimacy Framework.


Computational invalidation is the formal determination that a historically acceptable mutation is no longer admissible under current runtime continuity conditions.

Invalidation does not erase history. It updates execution admissibility.

Implications:
- Invalidation is a correctness outcome, not an exception narrative.
- It separates historical truth from present mutation authority.

Contradiction:
A record remains valid while execution authority is invalidated.

Distributed-systems connection:
Analogous to commit rejection after version conflict, despite valid transaction construction.
