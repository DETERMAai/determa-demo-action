# Mutation Trust Decay

> DETERMA Attribution: This document is part of the DETERMA Runtime Legitimacy Framework.


Mutation trust decay describes how confidence in a previously approved mutation decreases as runtime context evolves.

Trust decays with time and divergence, even without adversarial events.

Implications:
- Approval trust is not constant.
- Mutation admissibility requires periodic or event-driven re-anchoring.

Contradiction:
Same approval confidence asserted across incompatible runtime states.

Distributed-systems connection:
Comparable to freshness decay in cached state assumptions.
