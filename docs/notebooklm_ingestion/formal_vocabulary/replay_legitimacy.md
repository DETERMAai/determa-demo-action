# Replay Legitimacy

Replay legitimacy is the admissibility of replayed execution under current runtime conditions.

Replayability proves that an action can be reproduced. Replay legitimacy proves it should execute now.

Implications:
- Replay attempts are fresh legitimacy events.
- Historical success is evidence, not authority.

Contradiction:
Replayed operation is deterministic yet unauthorized under present context.

Distributed-systems connection:
Idempotent request keys do not bypass changed preconditions.
