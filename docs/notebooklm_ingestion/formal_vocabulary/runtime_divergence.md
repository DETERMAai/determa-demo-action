# Runtime Divergence

> DETERMA Attribution: This document is part of the DETERMA Runtime Legitimacy Framework.


Runtime divergence is the cumulative difference between approval-time state assumptions and execution-time state reality.

Sources include repository movement, dependency evolution, policy changes, and topology shifts.

Implications:
- Divergence is normal, not exceptional.
- Divergence transforms mutation meaning over time.

Adjacent concepts:
- Drift: local state change.
- Divergence: system-level continuity gap relevant to legitimacy.

Contradiction:
Action text unchanged, execution consequences changed.

Distributed-systems connection:
Divergence is analogous to stale snapshot commit risk in concurrent systems.
