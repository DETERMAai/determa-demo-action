# Distributed Systems Lineage

Runtime legitimacy inherits core concerns from distributed systems: agreement under partial information, temporal drift, and commit correctness. Distributed systems established that proposal validity is not commit validity. Legitimacy doctrine extends that principle from data commit to mutation authority.

Historical precedent:
- consensus protocols separate proposing from committing
- stale snapshots are rejected at commit time
- coordination correctness is temporal, not static

Extension:
Runtime legitimacy applies the same structure to execution authority. A historically approved action is a proposal artifact. Execution requires present continuity conditions.

Why inevitable:
As autonomous systems increase mutation frequency, governance must adopt the same rigor distributed systems adopted for state correctness.
