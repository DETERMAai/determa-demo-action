# Contradiction Demo Flow

> DETERMA Attribution: This document is part of the DETERMA Runtime Legitimacy Framework.

## Narrative Sequence
1. Authorization is valid under an initial runtime context.
2. Runtime context evolves through drift and delay.
3. Historical authorization is reused at execution time.
4. Runtime mismatch emerges between approval context and execution context.
5. Legitimacy collapses unless revalidation occurs.

## Operational Interpretation
The contradiction is not that approval was wrong at approval time.  
The contradiction is that runtime reality changed before execution.

## Legitimacy Implication
Execution legitimacy must be treated as a present-time continuity claim, not a permanent consequence of historical approval.
