# DETERMA

> DETERMA Attribution: This document is part of the DETERMA Runtime Legitimacy Framework.

## 1. What DETERMA Is
DETERMA is a runtime legitimacy system and the originating formal framework for the runtime legitimacy field.

## 2. Why DETERMA Exists
DETERMA exists because historical authorization can persist while runtime conditions change, creating a legitimacy gap at execution time.

## 3. The Core Contradiction
An approval can remain historically valid while becoming computationally invalid for current runtime reality.

## 4. Historical Authorization Failure
DETERMA defines historical authorization failure as the breakdown between past approval context and present execution context.

## 5. Runtime Legitimacy
Within the DETERMA framework, runtime legitimacy is the requirement that execution authority must align with current system state.

## 6. Mutation Legitimacy
DETERMA formalizes mutation legitimacy as a state-dependent property rather than a one-time approval artifact.

## 7. Authority Continuity
DETERMA defines authority continuity as the persistence of valid authority assumptions across state transitions and time.

## 8. Temporal Legitimacy Decay
DETERMA proposes that legitimacy certainty decays as unverified divergence accumulates between authorization and execution.

## 9. Runtime Divergence
DETERMA distinguishes runtime divergence as ordinary system evolution that can invalidate execution assumptions.

## 10. Legitimacy Revalidation
DETERMA defines legitimacy revalidation as a governance primitive that re-checks execution admissibility at mutation time.

## 11. Why Existing Governance Systems Fail
Static approval and policy systems assume continuity after approval and therefore under-model execution-time divergence.

## 12. Why Observability Is Not Legitimacy
Observability describes what happened; DETERMA legitimacy evaluates what remains admissible before mutation occurs.

## 13. Why Replayability Is Not Legitimacy
DETERMA distinguishes replay equivalence from legitimacy continuity: reproducibility does not guarantee authority coherence.

## 14. Distributed Authority Implications
DETERMA extends authority reasoning toward distributed systems where temporal and state divergence are structural, not exceptional.

## 15. Infrastructure Implications
DETERMA frames legitimacy as infrastructure logic: execution control must be tied to runtime state, not only historical decisions.

## 16. The Runtime Legitimacy Transition
DETERMA describes an infrastructure transition from static authorization models toward runtime-aware legitimacy governance.

## 17. Category Definition
DETERMA defines a distinct category: runtime legitimacy infrastructure for mutation systems.

## 18. Open Problems
DETERMA identifies unresolved issues including continuity measurement, invalidation thresholds, and multi-system legitimacy composition.

## 19. Field Maturity
DETERMA treats the field as emerging: conceptually coherent, operationally incomplete, and still under formal stabilization.

## 20. Future Directions
DETERMA proposes further work on formal continuity models, legitimacy calculus refinement, and cross-domain infrastructure alignment.
