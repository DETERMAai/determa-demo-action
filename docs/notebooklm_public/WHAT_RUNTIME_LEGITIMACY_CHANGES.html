What Runtime Legitimacy Changes

Consequences of moving from static approval to runtime-aware execution legitimacy.

The approval remained. The runtime changed. Execution legitimacy diverged.

Authorization Alone Becomes Insufficient

Approval history remains important, but it no longer guarantees that execution still belongs to the current system reality.

Runtime State Becomes Central

Execution legitimacy depends on present runtime continuity: code state, dependencies, and execution context.

Replayability Fails As A Proxy

A replayable artifact can still be computationally illegitimate when runtime assumptions have drifted.

Evidence Becomes Operational

Append-only evidence moves governance from debate to reconstruction and deterministic evaluation.

Governance Surface Shifts

Controls move from one-time approval checkpoints to execution-time legitimacy revalidation.
