# Legitimacy Field Invariants

These invariants are intended to hold across distributed systems, runtime governance, mutation systems, replay systems, and authority systems.

1. Authority validity is state-dependent.
2. Historical approval cannot guarantee future legitimacy.
3. Legitimacy weakens under unverified divergence.
4. Execution authority must be evaluated at execution boundary.
5. Replayability and legitimacy are distinct properties.
6. Identical mutation syntax can carry non-identical runtime meaning.
7. Delay increases legitimacy uncertainty.
8. Runtime continuity is required for authority continuity.
9. Externalized authority boundaries reduce self-legitimization risk.
10. Invalidation preserves historical truth while refusing present inadmissibility.
