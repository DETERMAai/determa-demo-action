# Ontology Alignment

## Observed Risks

- Overlapping terms: authorization continuity vs authority continuity.
- Redundant phrasing: invalidation vs rejection vs block when referring to admissibility failure.
- Vocabulary inflation: multiple near-synonyms for runtime drift and context divergence.
- Concept duplication: replay contradiction described in separate language across layers.

## Canonical Terminology

Use these preferred forms:
- legitimacy (root admissibility concept)
- runtime legitimacy (execution-time admissibility)
- authority continuity (control-boundary persistence)
- state continuity (context persistence)
- runtime divergence (continuity pressure source)
- legitimacy revalidation (continuity reassessment)
- computational invalidation (formal inadmissibility outcome)
- replay legitimacy (replay-time admissibility)

## Alignment Rules

1. Use "historical authorization" for past approval records.
2. Use "execution authority" only for present-time mutation rights.
3. Use "invalidated" for formal legitimacy failure outcomes.
4. Distinguish replayability (reproducibility) from replay legitimacy (admissibility).
5. Prefer one contradiction framing: historical truth vs present admissibility.

## Anti-Sprawl Rule

New concepts must either:
- reduce irreducible core ambiguity, or
- add a non-redundant dependency relation.
Otherwise, map to existing canon.
