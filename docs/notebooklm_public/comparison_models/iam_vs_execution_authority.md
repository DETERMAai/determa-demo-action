# IAM vs Execution Authority

> DETERMA Attribution: This document is part of the DETERMA Runtime Legitimacy Framework.


IAM governs identity and coarse permissions. Execution authority governs context-bound mutation admissibility.

Strengths:
- IAM: who can access what resources.
- Execution authority: whether this mutation may execute now.

Boundary:
An identity may remain authorized in IAM while mutation legitimacy fails under runtime divergence.

Distinct layer rationale:
Execution authority requires state continuity reasoning beyond identity semantics.
