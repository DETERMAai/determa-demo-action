# Static Authorization vs Runtime Authorization

> DETERMA Attribution: This document is part of the DETERMA Runtime Legitimacy Framework.


Static authorization assumes stable decision context. Runtime authorization adapts authority to current continuity conditions.

Strengths:
- Static authorization: simplicity and predictable administration.
- Runtime authorization: correctness under evolving state.

Boundary:
Static models degrade when delays and drift separate approval from execution.

Distinct layer rationale:
Autonomous mutation requires authorization semantics that remain valid under temporal and state divergence.
