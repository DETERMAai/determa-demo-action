# Execution Authority

Execution authority is the present-time right to mutate external system state under current continuity constraints.

It is not equivalent to model capability, role identity alone, or historical approval status.

Implications:
- Authority must be runtime-bound.
- Authority claims must survive state and temporal checks.

Contradiction:
High-confidence mutation proposal treated as sufficient authority proof.

Distributed-systems connection:
Comparable to commit privilege requiring present protocol conditions.
