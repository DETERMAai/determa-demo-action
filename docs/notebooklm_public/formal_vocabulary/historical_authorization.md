# Historical Authorization

> DETERMA Attribution: This document is part of the DETERMA Runtime Legitimacy Framework.


Historical authorization is the recorded approval state of an action under prior runtime conditions.

It is essential for accountability, but insufficient as a stand-alone execution predicate.

Implications:
- Historical authorization should be preserved.
- Its use for execution must be continuity-conditioned.

Contradiction:
Treating historical record truth as present authority truth.

Distributed-systems connection:
Equivalent to confusing log history with current commit admissibility.
