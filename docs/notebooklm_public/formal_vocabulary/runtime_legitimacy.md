# Runtime Legitimacy

Runtime legitimacy is legitimacy evaluated at execution time against current runtime state.

It answers: does this mutation still belong to the system reality that now exists?

Runtime legitimacy differs from approval-time confidence. Approval captures a judgment under earlier conditions; runtime legitimacy tests whether those conditions still hold.

Implications:
- Execution can be blocked without invalidating historical approval.
- Runtime checks are correctness boundaries, not ceremony.

Contradiction:
Same mutation identity, different runtime state, unresolved legitimacy.

Distributed-systems connection:
Equivalent to commit-time validation after concurrent state movement.
