# Governed Autonomous Execution

Governed Autonomous Execution means autonomous systems can propose and execute changes only under Runtime Authority controls.

## Core Separation

- Intelligence proposes.
- Runtime Authority governs mutation admissibility.

This separation prevents capability from being treated as authority.

## Operational Model

1. Intent is generated.
2. Authority is validated for current context.
3. Runtime Legitimacy is evaluated.
4. Mutation is either admitted with scoped authority, or denied fail-closed.
5. Immutable Authority Lineage records outcome.

## DATP Contribution

DATP provides the deterministic authority transition path from intent to constrained execution.
It does not replace orchestration systems; it constrains mutation authority across runtime change.
