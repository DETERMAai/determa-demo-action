# Runtime Legitimacy

Runtime Legitimacy answers a specific question:

Is this execution still admissible under current runtime reality?

## What It Is

Runtime Legitimacy is an execution-time authority check.
It compares current runtime conditions against the authority assumptions that supported prior approval.

## Why It Matters

Autonomous execution often runs across:

- delayed timelines
- dependency drift
- environment mutation
- replayed authority artifacts

Under these conditions, historical approval and present legitimacy can diverge.

## Relationship To DATP

Within DETERMA, DATP operationalizes Runtime Legitimacy by requiring Authority Continuity before mutation.
If continuity fails, execution fails-closed.

## Canonical Terms

- Runtime Legitimacy
- Authority Continuity
- Mutation Admissibility
- Runtime Divergence
- Runtime Authority
