# Mutation Legitimacy Walkthrough

Conceptual walkthrough:
1. AI proposes a mutation.
2. Human approval is granted under current runtime conditions.
3. Time passes and runtime reality evolves.
4. Execution is resumed using historical authorization.
5. Runtime legitimacy is re-evaluated at execution time.
6. If runtime context no longer aligns, mutation legitimacy is invalidated.

Result:
- execution decisions remain aligned with current runtime state.
